    • 事件监听

      目标:能够给dom元素添加事件监听

        什么是事件?
      • 事件是在编程时系统内发生的动作或发生的事情，比如用户在网页上单击一个按钮
        • 什么是时间监听
      • 就是让程序检测是否有事件产生，一旦有事件触发，就立即调用一个函数做出响应，也称为绑定事件或注册事件
        比如鼠标经过时显示下拉菜单，比如点击可以播放轮播图等等
        • 语法
        元素对象.addEventListener('事件类型',要执行的函数) //注意事件类型一定要加上引号
        事件监听三要素
        • 事件源:哪个dom元素被事件触发了，要获取dom元素
        • 事件类型:用什么方式触发，比如鼠标点击，鼠标经过等
        • 事件调用的函数:事件触发时要做什么

    • 拓展-事件监听版本

      • DOM L0
        事件源.on事件
      • DOM L2
        事件源.addEventListener(事件，时间处理函数)
      • 区别

        on方式会被覆盖，addEventListener方式可以绑定多次，拥有事件更多特征，推荐使用

      • DOM L3

        DOM 3级时间模块在DOM2级时间的基础上重新定义了这些事件，也添加了一些新事件类型

    事件对象

    //语法：如何获取 >在事件绑定的回调函数的第一个参数就是事件对象 >一般命名为event、evt、ev、e 元素.addEventListener('click',function(e){ })

    • 获取事件对象

      • 事件对象是什么?
        • 也是个对象，这个对象里有事件触发时的相关信息
        • 例如:鼠标点击事件中，事件对象就存了鼠标点在哪个位置等信息
      • 使用场景
        • 可以判断用户按下哪个案件，比如回车键可以发布新闻
        • 可以判断鼠标点击了那个元素，从而做出相应的操作

    • 事件对象常用的属性

      部分常用属性
      • target:获取当前事件信息
      • type:获取当前的事件类型
      • clientX/clientY:获取光标相对于浏览器在可见窗口左上角的位置
      • offsetX/offsetY:获取光标相对于当前dom元素左上角的位置
      • key: 用户按下的键盘键的值。现在不提倡使用keyCode

    环境对象

    目标:能够分析判断函数运行在不同环境中this(它代表当前函数运行时所处的环境)所指代的对象

    环境对象:指的是函数内部特殊的变量this,它代表着当前函数运行时所处的环境

    利用环境对象可以使代码更加简洁

    函数的调用方式不同，this指向的对象也不同，直接调用函数，那么其实是相当于调用window函数

    [谁调用，this就是谁]这是判断this指向的粗略规则

    回调函数

    什么是回调函数?

    如果将函数A作为参数传递给B时，我们称函数A为回调函数，即函数作为参数被调用

    //一个常见的使用场景即回调函数 function fn(){} //fn传递给了定时器，此时fn就是回调函数 setInterval(fn,1000)
    //还有我们学过的事件监听，本质上后一个匿名函数也是个回调函数 box.addEventListener('click',function(){})
